# Business Motivation Model (BMM) – DataVision s.r.o.

**Účel:** praktický úkol 3 – strukturovaný model dle **OMG Business Motivation Model** (zjednodušená aplikace pro školení).  
**Vstupy:** případová studie, `PEST_analyza_DataVision.md`, `Porter_5_sil_DataVision.md`, `SWOT_DataVision.md`  
**Firma:** DataVision s.r.o. — datová analytika, BI, integrace, vývoj aplikací, školení (~25 zaměstnanců, Brno)

---

## 1. Ends (cílový stav)

### 1.1 Vision (vize)

**Vize:** Být v **České republice (a vybraných projektech v regionu)** spolehlivým partnerem středních firem pro **datově řízené rozhodování** — integrace dat z ERP/CRM, **kvalitní reporting a BI**, bezpečné **on-prem a hybridní** dodávky, s důrazem na **odbornost a dlouhodobé vztahy** místo masové „objemové“ produkce.

### 1.2 Goals (cíle – obecné směry)

| ID | Cíl | Popis |
|----|-----|--------|
| G1 | **Udržitelný růst** | Růst obratu a marže bez zadlužení a bez ztráty kvality. |
| G2 | **Diferenciace hodnotou** | Být vnímán jako partner pro integrace, kvalitu dat a governance, ne jen jako „dodavatel dashboardů“. |
| G3 | **Odolnost vůči trhu** | Snížit závislost na jedné části trhu (např. jen jednoduché BI) a na několika klíčových lidech. |
| G4 | **Compliance a důvěra** | Plnit požadavky GDPR, bezpečnosti a auditovatelnosti u klientů s citlivými daty. |

### 1.3 Objectives (konkrétní měřitelné cíle – příklady pro 3–5 let)

*V praxi doplnit konkrétní čísla dle strategie firmy; zde ilustrativní formulace.*

| ID | Objective | Měřítko (příklad) |
|----|-----------|-------------------|
| O1 | Zvýšit podíl zakázek z **integrací, datové kvality a governance** | % tržeb z těchto kategorií / počet projektů |
| O2 | Zvýšit **opakované zakázky** u stávajících klientů | % tržeb z follow-up / retence |
| O3 | Snížit **závislost na 3 klíčových osobách** (dokumentace, sdílení rolí) | rizikový index / počet kritických rolí |
| O4 | Udržet **finanční stabilitu** (bez dluhů, zdravá marže) | ukazatele z účetnictví |
| O5 | **Viditelnost odbornosti** (webináře, case studies) | počet publikací / návštěvnost |

## 2. Means (prostředky k dosažení cílů)

### 2.1 Strategies (strategie)

| ID | Strategie | Napojení na cíle |
|----|-----------|------------------|
| S1 | **Specializace na segment** (výroba, logistika, menší finanční instituce) | G2, O1 |
| S2 | **Posun nabídky „nahoru“** — složitější integrace, bezpečnost, DS, školení | G2, G3, O1 |
| S3 | **Prodej přes reference a vztahy** + cílená odborná viditelnost | G1, O2, O5 |
| S4 | **Řízení kapacity** — výběr zakázek, partneři pro špičky, produktizace opakujících se částí | G1, G3, O3 |
| S5 | **Hybridní a on-prem kompetence** jako konkurenční výhoda u konzervativních klientů | G4, diferenciace |

### 2.2 Tactics (taktiky – konkrétní kroky)

| ID | Taktika | Poznámka |
|----|---------|----------|
| T1 | Zavést **šablony** pro governance a bezpečnostní aspekty BI projektů | opakovatelnost, nižší náklady |
| T2 | **1× měsíčně** odborný výstup (LinkedIn / webinář / krátký článek) | O5 |
| T3 | **Smluvní limity** odpovědnosti a selekce tendrů | ochrana marže a rizik |
| T4 | **Mentoring a dokumentace** u kritických řešení | O3 |
| T5 | Rozšířit **školení** (Power BI, práce s daty) jako samostatnou nabídku | G2, příjem |
| T6 | Mapovat **ESG/datové požadavky** u stávajících výrobních klientů | nové projekty |

## 3. Influencers (vlivy)

### 3.1 Externí vlivy

| ID | Vliv | Zdroj (analýza) |
|----|------|-----------------|
| EX1 | Konkurence a substituty (freelance, low-code, integrátoři) | Porter, PESTLE |
| EX2 | Regulace (GDPR, NIS2) | PESTLE (P/L) |
| EX3 | Ekonomika klientů (investice do IT, inflace) | PESTLE (E) |
| EX4 | Trh práce (mzdy IT, nedostatek lidí) | PESTLE (E/S), Porter (dodavatelé) |
| EX5 | Technologie (AI, cloud, Microsoft roadmap) | PESTLE (T) |
| EX6 | Očekávání klientů (rychlost, kvalita dat) | PESTLE (S) |

### 3.2 Interní vlivy

| ID | Vliv |
|----|------|
| IN1 | Omezené kapacity (~25 lidí) |
| IN2 | Závislost na klíčových osobách |
| IN3 | Silné reference a síť kontaktů |
| IN4 | Stabilní finance, bez dluhů |
| IN5 | Slabší systematický marketing |
| IN6 | Portfolio technologií (Power BI, SQL, Python, Docker, Azure) |

## 4. Assessment (hodnocení vlivů)

*U významných vlivů: příležitost (O) / hrozba (T) / obojí a **dopad na DataVision**.*

| Vliv | O / T | Dopad (stručně) |
|------|-------|-----------------|
| EX1 – konkurence | T | Tlak na cenu a substituce jednoduchých zakázek |
| EX2 – regulace | O+T | Poptávka po konzultacích; náklady compliance |
| EX3 – ekonomika klientů | T | Odklady projektů při škrtech |
| EX4 – trh práce | T | Vyšší náklady, obtížný nábor |
| EX5 – AI / cloud | O+T | Nové služby vs. tlak na rutinní práci |
| EX6 – očekávání | T | Tlak na termíny a přesnost dat |
| IN1 – kapacity | T | Omezuje růst bez efektivity |
| IN2 – klíčoví lidé | T | Riziko kontinuity |
| IN3 – reference | O | Konkurenční výhoda v prodeji |
| IN4 – finance | O | Odolnost v horších letech |
| IN5 – marketing | T | Nižší viditelnost vůči novým hráčům |
| IN6 – technologie | O+T | Shoda s trhem; závislost na vendorovi |

## 5. Directives (řízení)

### 5.1 Policies (zásady)

| ID | Zásada |
|----|--------|
| P1 | **Kvalita a auditovatelnost** datových řešení má přednost před „rychlým“ dodáním bez dokumentace. |
| P2 | **Bezpečnost a ochrana dat** klienta musí být posouzena u každého projektu s osobními nebo citlivými daty. |
| P3 | **Individuální přístup** a referenční vztahy jsou součástí obchodního modelu. |
| P4 | **Neakceptovat** zakázky s neudržitelným SLA nebo nejasnou odpovědností bez řízení změn. |
| P5 | **Investice do znalostí** týmu (školení, sdílení know-how) jsou systémová povinnost. |

### 5.2 Business Rules (obchodní pravidla – příklady)

| ID | Pravidlo |
|----|----------|
| BR1 | Každý projekt nad definovaným prahem (např. X MD) musí mít **písemné zadání a analýzu** (scope). |
| BR2 | Výstupy musí být **smluvně sladěny** s požadavky na GDPR / přístupová práva u BI. |
| BR3 | **Změny rozsahu** musí projít odsouhlasením (e-mail / ticket) a aktualizací odhadu. |
| BR4 | Kritická řešení musí mít **zálohu znalosti** (druhá osoba nebo dokumentace). |
| BR5 | Nasazení u klienta (Docker/on-prem) musí mít **předaný provozní postup** a kontaktní osobu. |

## 7. Shrnutí (pro závěr úkolu)

Model BMM pro DataVision spojuje **vizi datově řízeného partnera pro SME** s **cíli růstu a diferenciace** a s **konkrétními strategiemi** (segment, hodnotová nabídka, kapacita, reference). **Vlivy** z PESTLE a Portera jsou v **Assessment** zařazeny jako příležitosti a hrozby; **Policies a Business Rules** zajišťují konzistenci s regulací a riziky malé firmy. Tím je splněn záměr úkolu 3: **proč firma existuje, kam směřuje, co ji ovlivňuje a jak na to reaguje**.
